Glossary

A

Administrator

An Administrator is able to add and retire users, create filters and define the default attributes of the models.

Assistant

An Assistant is a reporting relationship that is intended to represent it as not being superior to the Nodes beneath it within the chart.

Attributes

The Attributes are the characteristics of the chart and its Nodes. These include the size of each Node and the colours that are used within the chart.

B

Branding

Branding allows for the colours and images that your organisation has adopted to be applied to both the appearance of Vivichart and the charts that are published.

C

Chart Area

The Chart Area is the portion of the screen that displays.

A Chart Builder can

  • move information from a Selection List into this area to create Nodes, and
  • move information back from the Chart Area to the Selection List upon either deleting a Node or making one Vacant.

 

 

Chart Builder
A Chart Builder is a User who is able to create charts.

Child
A Child is a Node that reports to its Parent Node.

Collapsed or Expanded State

The collapsed or expanded state of a Node determines whether its child Nodes are visible or not.

 

 

 

Co-Parent
A Co-Parent is a group of one or more Nodes which may have both a Parent and Child Nodes.

D

Data Node
A Data Node is a Node that has been populated with information from a Data View.

Data Source

A Data Source is the information within a spreadsheet’s worksheet or another file of data that has been imported into ViviChart.

Data Type
A Data Type may be one of either text, date or a number.

Data View

A Data View is the group of Fields that have been selected from a Data Source to appear within the lines of a Node.

Domain Name
A domain name is the suffix that identifies a client’s organisation. For example, businessname.com is the domain name for john.smith@businessname.com.

Drag and Drop

Drag and drop allows the user to both

  • ‘drag’ an item from a Selection List and drop it within the chart in order to add a Node, and
  • 'drag' a Node (and its children) and drop it onto another Node in order to change its reporting relationship.

 

E

eMail Upload

eMail Upload allows for Data Sources to be created and updated upon Vivichart receiving eMails that contain one or more attachments that provide the relevant information.

F

Field

A Field is a unique description of the rows of information within a Data Source. For example, 'Forename 1', 'Surname', 'Job'.

 

File Formats

Information can be imported into Vivichart in the following File Formats – Excel spreadsheets, CSV, XML and jSON.

Filter

A Filter is used to limit the rows of information that appear within a Selection List when a user is creating or updating a chart. For example, ‘employees’ or ‘positions’ in a single or range of 'departments'.

G

Group Node

A Group Node  is a Node for which lines of text have been entered manually in order to group other Data and Group Nodes beneath it.

 

H

HEX
HEX is a method to precisely define a colour, amongst a list of just under 17 million. For example, Black Is expressed as #000000, yellow as #FFFF00 and White as #FFFFFF

I

Image

An Image can be displayed within each Data Node of those charts that have been set to display this information. The physical size of each Image will be reduced automatically to 10,240 bytes  if it exceeds this size when it is uploaded.

L

Level

A Level within a hierarchical chart signifies the relative position of its Nodes to the top level/first Node.

N

Node

A Node is a container within a chart that is either

  • populated with information from the Selection List (a “Data Node”),

          

  • set to being Vacant (a "Vacant Node"), or

           

  • displayed with text that has been entered manually in order to group other Nodes (a “Group Node”).

          

 

O

Occurrence of a Data Source
Each Occurrence of a Data Source  can be defined as being unique based upon its date (‘04/03/2022’), its date and time (‘12:31 04/03/2022’) or the nearest hour rounded down (‘12:00 04/02/2022’) or rounded up (‘13:00 04/02/2022’).  Where an occurrence overlaps an existing one , the Data Source can be defined to either reject the new occurrence or update the existing one with the new information

P

Parent

A Parent is the Node to which a Child Node reports to.

Partial text search

A partial text search will limit the Selection List to display only those rows that contain the character or characters that have been entered within the corresponding Search box.

Primary Data Source

A Primary Data Source is one within which each row of information contains a unique reference. For example, an ‘employee reference’ or ‘position reference’.

Published

A Published chart is one that its Chart Builder has made available to be viewed by other persons.

R

Related Data Source

A Related Data Source is one that shares the unique reference of a Primary Data Source and is used to list related information. For example, a list of an employees’ jobs would contain the ‘employee reference’ as well as information about each job.

Reporting Relationship

A Reporting Relationship is a line that illustrates the association between a Parent and its child(ren).

 

S

Selection List

A Selection List is the information from the Occurrence of a Data Source that is displayed to a user in order that the user can create Data Nodes within a chart and establish reporting relationships.

 

 

Service Level

The Service Level determines the maximum number of users that can create charts

Shadow

The Shadow effect accentuates a Node's border.

 

Sibling

A Sibling is a Node with the same Parent as another Node.

 

 

Software-as-a-Service

Software-as-a-Service is a software licensing and delivery model in which software is licensed on a subscription basis and is hosted centrally.

T

Tooltip

A Tooltip is the descriptive text that is displayed when the cursor is placed over a Node.

U

Unique Information
Unique Information is the data that is held within a Field that specifically identifies a row/record of a Data Source. For example, an 'employee reference' might identify 'John Brown' and all the other information that is held for him.

V

Vacant

A Vacant Node is a generic term that is used to signify that a Data Node has not been populated from the Selection List.

This name may be changed for different Data Sources. For example, ‘Expired’, ‘Vacant’, ‘Out of Use’, etc.

Z

ZIP file

A ZIP file is an archive file format that compresses one of more other files. Any ZIP file may be encrypted by requiring that a password is entered before its content can be seen.
