vivichart is a cloud based, data-driven software-as-a-service that allows for professional-looking graphical charts to be created and kept up to date extremely easily within a web browser environment, and from information that has been submitted as spreadsheets and in other file formats. This information can be uploaded both directly and upon vivichart receiving file attachments (including encrypted and unencrypted ZIP files) within eMails.

 

A person with a limited knowledge of Microsoft Excel has the capacity to start using vivichart immediately and without any prior training. vivichart does not require that anyone in your organisation becomes proficient in the use of a particular graphics tool in order to produce charts, and there is no need for a ‘skills transfer’ should staff leave or be redeployed . Upon vivichart being informed of the ‘replacement’ user, it will send a detailed explanation to that person by eMail as what needs to be done.

 

The charts may be published and presented with your organisation’s own branding on the internet, within an intranet and by email. If your charts are to be created by many users, the display characteristics and the type of information that is displayed may be predefined by an Administrator to ensure uniformity and make the chart creation process even easier for other users.

 

When viewed, the collapsed or expanded state of the Nodes within the chart can be adjusted in order to see more or less information. Charts may also be searched to highlight those Nodes that match the criterion that has been expressed.

 

Upon existing information being updated, vivichart examines all of the current models to determine whether they need to be updated, and will notify each user, who has created a model based upon this information, by email of the changes that may need to be implemented.  For example, models based upon ‘employees’ might need changing if one or more people have left, joined or moved department.  Where the source of a Node’s information no longer exists, the Node itself can be set to being Vacant automatically.  In such circumstances, a user may respond by email in order to update the effected models.

 

Historical charts are retained on-line in order that changes to them can be visualised very clearly.
